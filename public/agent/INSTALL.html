ThermaShift Monitoring Agent

Installation & Setup Guide — v2.0

This agent collects thermal and performance data from your servers and sends it to the ThermaShift Thermal Intelligence Platform for real-time monitoring, AI-powered analysis, and cooling optimization recommendations.

Requirements

RequirementDetails
Python 3.7+Pre-installed on most Linux and macOS. Download for Windows.
psutil (Python package)Required. Collects CPU, memory, disk, and temperature data.
GPUtil (optional)For GPU temperature monitoring (NVIDIA GPUs only).
Network accessHTTPS outbound to supabase.co for data delivery.

One-Line Install Linux

curl -sSL https://thermashift.net/agent/install-linux.sh | bash

Manual Install

1
Verify Python is installed 
python3 --version

If not installed:

# Debian / Ubuntu
sudo apt update && sudo apt install python3 python3-pip

# RHEL / CentOS / Fedora
sudo yum install python3 python3-pip

# Arch Linux
sudo pacman -S python python-pip
2
Install Python dependencies 
pip3 install psutil

# Optional: GPU monitoring (NVIDIA only)
pip3 install GPUtil
3
Download the agent 
mkdir -p ~/.thermashift
curl -sSL https://thermashift.net/agent/thermashift-agent.py -o ~/.thermashift/thermashift-agent.py
chmod +x ~/.thermashift/thermashift-agent.py
4
Run the setup wizard 
python3 ~/.thermashift/thermashift-agent.py --setup

Follow the prompts to configure your facility name, rack ID, polling interval, and data delivery.

5
Start monitoring 
# Run in foreground (for testing)
python3 ~/.thermashift/thermashift-agent.py --daemon

# Run in background
nohup python3 ~/.thermashift/thermashift-agent.py --daemon &

# Or set up as a systemd service (see below)

Run as a systemd Service (Recommended for Production)

# Create service file
sudo tee /etc/systemd/system/thermashift-agent.service > /dev/null << 'EOF'
[Unit]
Description=ThermaShift Monitoring Agent
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=root
ExecStart=/usr/bin/python3 /root/.thermashift/thermashift-agent.py --daemon
Restart=always
RestartSec=30

[Install]
WantedBy=multi-user.target
EOF

# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable thermashift-agent
sudo systemctl start thermashift-agent

# Check status
sudo systemctl status thermashift-agent

Run as a Windows Service (Recommended for Production)

Use Task Scheduler for automatic startup:

  1. Open Task Scheduler (search in Start Menu)
  2. Click Create Basic Task
  3. Name: ThermaShift Agent
  4. Trigger: When the computer starts
  5. Action: Start a program
  6. Program: python
  7. Arguments: C:\Users\YourName\.thermashift\thermashift-agent.py --daemon
  8. Check "Run with highest privileges"
  9. Click Finish

Setup Wizard

The setup wizard (--setup) walks you through configuring the agent. Here's what each option means:

Facility Name

Your company or data center name. Example: Flexential Charlotte

Facility ID

A unique identifier that links this agent to your facility on the ThermaShift dashboard. Your ThermaShift administrator will provide this. Example: flexential-clt-01

Rack/Server Name

Identifies this specific machine. Defaults to the hostname. Example: A1, Rack-3-Server-12

Cooling Type

The cooling method used for this server:

  • Air Cooling — Standard CRAC/CRAH units
  • RDHX — Rear-Door Heat Exchangers
  • D2C — Direct-to-Chip liquid cooling
  • Immersion — Full immersion cooling
  • Hybrid — Mix of cooling methods

Polling Interval

How often the agent collects and sends data:

PresetIntervalBest For
Real-time30 secondsActive troubleshooting, demos
Frequent1 minuteHigh-density or mission-critical racks
Standard5 minutesNormal production monitoring
Conservative15 minutesLow-priority or many agents
Hourly60 minutesBaseline/trend data collection

Data Delivery

Where collected data is sent:

  • ThermaShift Cloud (recommended) — Automatically configured. Data appears on your ThermaShift dashboard within seconds.
  • Custom webhook — Send to your own API, Zapier, or other service.
  • Local only — Saves to CSV on the local machine. Export manually.

Data Collected

The agent collects the following data points at each polling interval:

MetricSourceNotes
CPU TemperatureSystem sensorsLinux: lm-sensors / sysfs. Windows: WMI. macOS: powermetrics.
GPU TemperatureGPUtil (NVIDIA)Requires optional GPUtil package
Power DrawEstimatedBased on CPU usage, core count, and TDP model
CPU UsagepsutilPercentage across all cores
Memory UsagepsutilPercentage and total GB
Disk UsagepsutilRoot partition percentage
Inlet TemperatureConfigurableDefault: 22°C. Override in setup if you have inlet sensors.
Outlet TemperatureCPU tempUses CPU temperature as proxy for outlet air temp
Hostname & OSSystemAutomatically detected

Troubleshooting

CPU Temperature shows N/A

Linux: Install lm-sensors: sudo apt install lm-sensors && sudo sensors-detect

Windows: WMI thermal zone access may require Administrator privileges. Run the agent as Administrator.

macOS: Run with sudo for full sensor access.

The agent will still work without temperature data — it uses safe defaults (35°C outlet).

Send failed: Connection error

Check that the server has outbound HTTPS access to supabase.co. Some firewalls or proxies may block this. Test with:

curl -I https://auqklthrpvsqyelfjood.supabase.co

Permission denied errors

The agent needs write access to ~/.thermashift/ for config and logs. On Linux/Mac:

chmod -R 700 ~/.thermashift

Agent stops after closing terminal

Use one of the service methods above (systemd, Task Scheduler, or launchd) for persistent monitoring. Or run with nohup:

nohup python3 ~/.thermashift/thermashift-agent.py --daemon > /dev/null 2>&1 &

Reset configuration

rm ~/.thermashift/config.json
python3 ~/.thermashift/thermashift-agent.py --setup

Security

  • Data is transmitted over HTTPS (TLS encrypted)
  • The agent collects system metrics only — no file contents, user data, network traffic, or credentials
  • Configuration and API keys are stored locally in ~/.thermashift/config.json
  • The agent runs with the permissions of the user that started it
  • No inbound ports are opened — the agent only makes outbound HTTPS requests
