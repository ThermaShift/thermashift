## PART 2: AUTOMATED POSTING WITH LINKEDIN API + MAKE.COM + CLAUDE API

This pipeline will: Generate posts with Claude API > Review/approve > Publish to LinkedIn — all automated on a schedule.

---

### Architecture Overview

```
[Schedule Trigger in Make.com]
        |
        v
[Claude API Module] -- generates post content
        |
        v
[Approval Step] -- (optional) email/Slack notification to review
        |
        v
[LinkedIn API Module] -- publishes to company page
        |
        v
[Google Sheets Module] -- logs what was posted (optional)
```

---

### STEP 1: Get LinkedIn API Access

1. Go to https://developer.linkedin.com/
2. Click **"Create App"**
3. Fill in:
   - **App name:** `ThermaShift Content Automation`
   - **LinkedIn Page:** Select your ThermaShift company page
   - **App logo:** Upload your logo
   - **Legal agreement:** Check the box
4. Click **Create app**

5. Go to the **"Products"** tab in your app settings
6. Request access to:
   - **"Share on LinkedIn"** — this gives you `w_member_social` permission (for posting from your personal profile)
   - **"Advertising API"** — NOT needed, skip
   - **"Community Management API"** — request this for posting to your **company page** (gives `w_organization_social`)

7. Go to the **"Auth"** tab:
   - Copy your **Client ID** and **Client Secret** (you'll need these in Make.com)
   - Add a redirect URL: `https://www.integromat.com/oauth/cb/linkedin` (Make.com's OAuth callback)

8. **Scopes you need:**
   - `w_organization_social` — post to company page
   - `r_organization_social` — read company page posts
   - `rw_organization_admin` — manage company page

> **Note:** LinkedIn's Community Management API requires your app to be verified. Submit verification — they typically approve consulting/marketing use cases within 1-3 business days.

---

### STEP 2: Set Up Claude API Access

1. Go to https://console.anthropic.com/
2. Create an account (or log in)
3. Go to **API Keys** and click **"Create Key"**
4. Name it: `ThermaShift-MakeAutomation`
5. Copy the key — you'll use it as a header in Make.com HTTP requests
6. Add billing: Claude API is pay-per-use. For post generation, expect ~$0.05-0.15 per post using Claude Haiku (cheapest) or ~$0.50-1.00 per post using Sonnet (higher quality).

**Recommended model for social posts:** `claude-haiku-4-5-20251001` — fast, cheap, great for short-form content.

### STEP 3: Build the Make.com Scenario

#### 3A: Create Account & Scenario

1. Go to https://www.make.com/ and create a free account
2. Click **"Create a new scenario"**
3. Name it: `ThermaShift LinkedIn Auto-Post`

#### 3B: Module 1 — Schedule Trigger

1. Add a **"Schedule"** module (it's the clock icon)
2. Set frequency: **Every week** on **Tuesday and Thursday at 9:00 AM EST**
   - (Optimal B2B LinkedIn posting times: Tue-Thu, 8-10 AM)
3. This is what kicks off the automation on schedule

#### 3C: Module 2 — Generate Post with Claude API (HTTP Module)

1. Add an **"HTTP > Make a request"** module
2. Configure:

| Field | Value |
|---|---|
| **URL** | `https://api.anthropic.com/v1/messages` |
| **Method** | `POST` |
| **Headers** | `x-api-key`: `YOUR_CLAUDE_API_KEY` |
| | `anthropic-version`: `2023-06-01` |
| | `content-type`: `application/json` |

3. **Request body** (set Body type to "Raw" and content type to "JSON"):

```json
{
  "model": "claude-haiku-4-5-20251001",
  "max_tokens": 1024,
  "messages": [
    {
      "role": "user",
      "content": "You are the content strategist for ThermaShift, a data center cooling optimization and waste heat recovery consulting firm. Our services: (1) Liquid cooling design & optimization, (2) Waste heat recovery & monetization, (3) AI-driven thermal intelligence monitoring, (4) ESG compliance & sustainability consulting.\n\nOur audience: Data center facility managers, VP of Operations at colocation providers, sustainability officers, and CTO/CTOs at hyperscale operators.\n\nWrite ONE LinkedIn post for our company page. Requirements:\n- 150-250 words\n- Open with a hook (surprising stat, bold claim, or contrarian take)\n- Provide genuine insight or actionable information (not just marketing fluff)\n- End with a soft CTA or thought-provoking question\n- Include 3-5 relevant hashtags at the end\n- Professional but not corporate — confident, direct, knowledgeable tone\n- Rotate topics across: cooling technology trends, waste heat economics, ESG regulatory updates, PUE optimization, liquid cooling adoption, sustainability ROI, industry news commentary\n- Do NOT use emojis\n- Do NOT start with 'Did you know' or 'Exciting news'\n\nReturn ONLY the post text, no preamble or explanation."
    }
  ]
}
```

4. **Parse the response:** Add a **"JSON > Parse JSON"** module after the HTTP request
   - Data structure: Create a new one with this schema:
   ```json
   {
     "content": [
       {
         "type": "text",
         "text": "string"
       }
     ]
   }
   ```
   - The post text will be at: `content[0].text`

#### 3D: Module 3 — (Optional) Approval via Email

If you want to review posts before they go live:

1. Add a **"Gmail > Send an email"** module (or any email module)
2. Send the generated post to yourself
3. Add a **"Webhooks > Custom webhook"** module that waits for your approval
4. In the email, include two links:
   - Approve: `https://hook.make.com/YOUR_WEBHOOK_ID?action=approve`
   - Reject: `https://hook.make.com/YOUR_WEBHOOK_ID?action=reject`
5. Add a **"Router"** after the webhook — one path for approve, one for reject

**Simpler alternative:** Skip the approval flow. Instead, set Make.com to run the scenario manually for the first 2 weeks so you can review before clicking "Run." Switch to auto after you trust the output quality.

#### 3E: Module 4 — Post to LinkedIn Company Page

1. Add a **"LinkedIn > Create a Company Page Share"** module
   - If this module isn't available, use **"HTTP > Make a request"** with the LinkedIn API directly:

| Field | Value |
|---|---|
| **URL** | `https://api.linkedin.com/v2/ugcPosts` |
| **Method** | `POST` |
| **Headers** | `Authorization`: `Bearer YOUR_LINKEDIN_ACCESS_TOKEN` |
| | `content-type`: `application/json` |
| | `X-Restli-Protocol-Version`: `2.0.0` |

**Body:**
```json
{
  "author": "urn:li:organization:YOUR_COMPANY_ID",
  "lifecycleState": "PUBLISHED",
  "specificContent": {
    "com.linkedin.ugc.ShareContent": {
      "shareCommentary": {
        "text": "{{POST_TEXT_FROM_CLAUDE_MODULE}}"
      },
      "shareMediaCategory": "NONE"
    }
  },
  "visibility": {
    "com.linkedin.ugc.MemberNetworkVisibility": "PUBLIC"
  }
}
```

**To find your Company ID:**
- Go to your LinkedIn company page
- Click "Admin tools" > "View as member"
- The URL will show: `linkedin.com/company/NUMBERS/` — those numbers are your Company ID

#### 3F: Module 5 — Log to Google Sheets (Optional but Recommended)

1. Add a **"Google Sheets > Add a Row"** module
2. Create a spreadsheet called "ThermaShift Content Log"
3. Columns: `Date`, `Post Text`, `Status`, `LinkedIn Post URL`
4. This gives you a record of everything posted

### STEP 4: Handle LinkedIn OAuth Token Refresh

LinkedIn access tokens expire every 60 days. To handle this:

**Option A (Easiest):** Use Make.com's built-in LinkedIn connection — it handles token refresh automatically. Go to Connections > Add > LinkedIn > Authorize.

**Option B (Manual):** If using HTTP modules directly:
1. Create a separate Make.com scenario that runs every 50 days
2. It calls LinkedIn's token refresh endpoint:
   ```
   POST https://www.linkedin.com/oauth/v2/accessToken
   grant_type=refresh_token
   refresh_token=YOUR_REFRESH_TOKEN
   client_id=YOUR_CLIENT_ID
   client_secret=YOUR_CLIENT_SECRET
   ```
3. Store the new token in Make.com's Data Store

**Recommendation:** Use Option A. Let Make.com handle it.

### STEP 5: Advanced — Content Variety System

To prevent repetitive posts, enhance the Claude prompt with a rotation system:

1. Add a **"Google Sheets > Get a Random Row"** from a "Topic Bank" spreadsheet:

| Topic | Angle | Key Stat |
|---|---|---|
| Liquid cooling adoption | ROI analysis | 40% energy savings |
| Waste heat recovery | Revenue opportunity | $2-5M/year for 10MW facility |
| ESG compliance | Regulatory pressure | SEC climate disclosure rules |
| PUE optimization | Quick wins | 0.1 PUE reduction = 10% energy savings |
| AI workload growth | Capacity planning | 120kW per GB200 rack |
| Water usage (WUE) | Sustainability | 660K gallons/day for 15MW facility |

2. Pass the random topic into the Claude prompt:
```
"Focus this post on: {{topic}} from the angle of: {{angle}}. Reference this stat if relevant: {{key_stat}}"
```

This ensures content variety across weeks.
